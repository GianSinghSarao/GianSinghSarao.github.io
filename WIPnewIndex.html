The String Encrypter

This project was the result of following a tutorial about a deprecated/obsolete tech stack while researching something else entirely. My original goal was to find a method to pack a web page into a single file that the user can download and use. Example applications would be HTML eBooks, and reference pages, but as a single file download. I came across HTA files and a tutorial on how to make a string encrypter. It seemed interesting so I followed the short tutorial to the end, then I began to customize it by putting in my nickname, and getting rid of the tacky marquee element usage.

The styling was light, to put it nicely, And frankly it didn't look modern or contemporary at all, like it could have. Though to be fair this is to be expected considering the short length of the tutorial/artice. Now this was a fairly rubbish application that relied on internet explorer being pre-installed and VBScript not being disabled. So I tried to improve it and failed. Mainly because converting VBscript to ES3 JavaScript then to ES5 was a non-trivial task for an amatuer like me who had only known modern ES5, and at the time didn't even know that JavaScript had versions. By now I had completely forgotten my original goal, which in hindsight had already been fulfilled by PDFs. I eventually gave in and started looking for a new solution to desktop HTML apps. I wanted something quick that could run on any os without dependending on IE, or IE specific behaviours. So I opened up chrome, and started coding. Just to be clear, I didn't open notepad or some other text editor. I just started writing HTML+CSS+JS in chrome's 'omnibox' albeit prefixed with a data url scheme, and without a single line break. At one point I pasted in an entire framework, and continued with that. I also pasted in a small symmetric enryption algorithm which i researched during a lesson at school and had saved in the url area of a bookmark which synced automatically to my laptop at home. And done. An entirely offline capable single file, single page application. I cut it out of the omnibox with CTRL-A and CTRL-X, and pasted it into an index.html file with CTRL-V. If someone right clicks the open webpage and clicks save page as, they can download a single file, and have it keep it's styles and functionality, even when offline. Now this was good. It was good but, I wasn't satisfied. After seeing the way I could launch an HTA as if it was an app, without any tabs or url bars, this wasn't good enough.
